9Časté chyby, kterých se Codex dopustí

1. Sub-nav vs Tabs. Editor má vertikální sub-nav (sticky), ne horizontální taby. Detail služby má horizontální taby. Tabulky v admin sekci mají vlastní tab pattern (ref_*).
2. Sticky save bar má 4 stavy. clean (skrytý) / dirty (warn) / saving (spinner) / error (bad).
3. Lifecycle select neukazuje disabled options. Skrytí, ne grey-out.
4. Service ID je trvale disabled po vytvoření. Zobrazte ho s background: #f1f5f9, ne jen disabled.
5. Requestable warning je banner, ne tooltip. Blokuje publish, povoluje draft.
6. Reviews kanban není drag-drop. Status se mění modal s rationale.
7. Decision modal preview shrnuje, co se zapíše do decision logu. Auditní transparence.
8. Pre-flight check v review modalu je side-effect-free GET. Volá se při otevření modalu.
9. Graph workspace má 3 sloupce. Toolbar vlevo, ne nahoře. Detail panel je collapsible.
10. RefTableEditor add-row je inline řádek nahoře tabulky, ne separátní form. Foreign key fields = typeahead select, ne plain dropdown.
11. Code editor má dark theme i když zbytek aplikace je light.
12. Sub-nav badges: červená (chybí povinné), oranžová (warning), zelená (sekce splněná dle readiness).
