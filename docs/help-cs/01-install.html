01 Instalace

Instalace, prvotní wizard a provozní předpoklady

Instalace připravuje databázi, první administrátorský účet, moduly a volitelná seed/import data. Je to jediná část aplikace, která je dostupná ještě před běžným přihlášením, proto musí být chráněna instalačním stavem a setup tokenem.

Teoretický smysl instalátoru

V enterprise nástroji není instalátor jen technický formulář. Určuje výchozí identitu instance, bezpečnostní režim, databázovou konektivitu, první účet s odpovědností a volbu modulů. Z pohledu ITIL je to přechod ze stavu deployment do stavu ready for operation. Z pohledu governance vytváří první auditovatelný základ, na kterém stojí správa služeb, capabilities a C3 evidence.

Instalační wizard: 12 obrazovek

Aktuální instalátor je průvodce přes 12 kroků. Není to jen série formulářů: každý krok snižuje jiné riziko. Nejdříve vysvětlí účel aplikace a vybere jazyk prostředí, potom ověří stav instance a setup token, nastaví identitu aplikace, zkontroluje secrety a databázi, vytvoří prvního administrátora, zapne moduly, připraví data, provede importní preview, spustí instalaci a nakonec zobrazí provozní summary.

01 Welcome

Popis aplikace, účel S3C Manageru a výběr CZE/ENG prostředí.

02 Setup token

Stav instance, setup token, start a reset instalačního běhu.

03 Configuration

Název, veřejná URL, timezone, storage path a HTTPS režim.

04 Secrets

Kontrola JWT, DB credentials a bezpečnostních předpokladů.

05 Admin

První lokální administrátor a pravidla hesla.

06 Connectivity

Ověření PostgreSQL, zápisu a platform schema.

07 Modules

Povinný Service Catalogue Core a volitelný C3 Taxonomy.

08 Data plan

Povinné reference, volitelná business data a demo data.

09 Upload

První nahrání CSV, XLSX nebo JSON souborů.

10 Preview

Klientské preview sloupců, řádků, chyb a warningů.

11 Execute

Finální zápis instalace, seed, import a log průběhu.

12 Summary

Výsledek, verze, schema, moduly a importní statistiky.

Detail jednotlivých obrazovek instalátoru

01

Welcome / popis aplikace a jazyk

route: /installpopis aplikaceCZE / ENG

První obrazovka slouží k orientaci. Uživatel vidí, k čemu S3C Manager slouží, proč aplikace existuje jako governance cockpit nad službami, capability, C3/FMN vazbami a readiness, a vybere si výchozí prostředí CZE nebo ENG. Setup token se zde nezadává, aby úvod nebyl bezpečnostně-technický hned v prvním kroku.

Co uživatel vidí

  • Stručný popis, že aplikace je service governance a rozhodovací cockpit.
  • Volbu prostředí CZE nebo ENG.
  • Tlačítko pokračovat na technický instalační krok.

Co obrazovka ovlivní

  • Nastaví locale cookie pro další kroky instalace.
  • Neprovádí zápis do databáze ani nespouští install lock.
  • Snižuje riziko, že admin začne instalaci bez pochopení účelu aplikace.

04

Secrets / bezpečnostní předpoklady

read-only checklistJWTDB credentials

Secrets obrazovka neslouží k zadávání hesel. Je to bezpečnostní kontrola, že citlivé hodnoty jsou nastavené mimo zdrojový kód a že se nebudou vypisovat do logů. Správně nastavené secrets jsou nutné pro autentizaci, audit a ochranu databáze.

KontrolaCo znamenáCo dělat při chyběDopad
JWT secret presentServer má tajný klíč pro podepisování tokenů.Doplnit JWT_SECRET nebo odpovídající secret v orchestraci.Bez něj nelze bezpečně přihlašovat uživatele.
DB credentials presentRuntime zná přístup k PostgreSQL.Opravit DB env/secrets a restartovat kontejnery.Bez DB credentials nelze číst ani zapisovat konfiguraci.
Outside source codeSecrets nejsou uložené v commitu nebo statickém konfiguračním souboru v repository.Přesunout je do `.env`, vaultu nebo orchestrátoru.Snižuje riziko úniku přístupů do Gitu.
No plaintext in logsAplikace nemá vypisovat hesla a tokeny do logů.Zkontrolovat log level a proxy middleware.Chrání provozní logy, support bundle a auditní exporty.

Admin / první administrátorský účet

Modules / výběr domén aplikace

Moduly určují, jaké části S3C Manageru budou aktivní. Core katalog je povinný, protože bez něj neexistují služby, portfolio ani governance základ. C3 modul je volitelný, ale pro FMN/NATO logiku, capability mapu a C3 entity workspace je zásadní.

VolbaStavCo zapneKdy použít

Preview / kontrola importu před zápisem

client-side previewcolumnswarningsno DB write

Preview krok je záměrně před Execute. Uživatel má vidět, kolik souborů a řádků bude importováno, které sloupce aplikace poznala, co chybí a zda jsou chyby fatální. Je to obdoba dry-run principu: rozhodnutí o zápisu má přijít až po srozumitelné kontrole.

PrvekVýznamKdy je problémDoporučená reakce

Rows to importPočet datových řádků rozpoznaných v souborech.Je 0 nebo výrazně nesedí s originálním exportem.Zkontrolovat oddělovač, hlavičku, formát nebo encoding.
ReadyPočet souborů bez fatální chyby.Ready je 0.Neprovádět Execute, opravit soubory.
ErrorsPočet fatálních chyb.CSV nemá datový řádek nebo se soubor nepodařilo načíst.Opravit zdrojový soubor, ne jen přejmenovat příponu.
Required present/missingRozpoznání povinných nebo očekávaných sloupců.Chybí identifikátor služby, název nebo zásadní mapping.Upravit exportní šablonu podle kapitoly 07 Import.
WarningsNe fatální upozornění, např. chybějící doporučené sloupce nebo service_type.Import projde, ale data budou hůř použitelná.Doplnit sloupce, pokud mají být služby governance-ready.
