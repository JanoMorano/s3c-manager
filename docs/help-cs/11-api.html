11 API

API parametry a integrační scénáře

API S3C Manageru je REST JSON rozhraní pod prefixem /api/v1. Slouží pro integrace s ITSM, CMDB, EA repository, Backstage, importními pipeline, reportingem a auditními exporty.

Teoretický význam API

API je smlouva mezi S3C Managerem a okolními systémy. V enterprise prostředí by API nemělo jen “nějak vracet data”. Musí držet stabilní identifikátory, role, audit, importní dry-run, exportní manifesty a jasné chování při chybách. Externí ITSM může řešit ticketing, CMDB může řešit CI/asset evidence a EA repository může držet modely; S3C Manager přes API poskytuje governance kontext služby, capability a C3 vazeb.

Procesní vazba: API podporuje ITIL integration mezi service catalogue, ITSM, CMDB a knowledge managementem. TOGAF/ArchiMate integrace využívá capability, application, data a technology vrstvy. Backstage export je vhodný pro developer portal, ale S3C zůstává governance cockpitem.

Integrační scénáře

ITSM integrace ITSM může odkazovat na request_channel_url, zakládat externí ticket a ukládat external_ticket_ref v external request logu. S3C drží governance context; ticket execution zůstává v ITSM.
CMDB integrace CMDB může dodávat service metadata, owner kontakty a vztahy. S3C není autodiscovery CMDB, ale umí číst/importovat governance metadata.
EA/ArchiMate integrace ArchiMate XML a C3 taxonomy endpoints propojují business procesy, aplikace, datové objekty, technology services a capability.
Backstage Export /api/v1/export/backstage/catalog-info poskytuje developer portal pohled, ale source of governance truth zůstává v S3C.
Reporting/BI Export manifest, bundle a read-only endpoints slouží pro reporting; zápisové integrace musí používat dry-run a audit.

Časté chyby integrací

ChybaDopadSprávný postup
Integrace používá admin účet pro čtení.Zbytečně velké riziko při kompromitaci.Použít viewer nebo dedikovaný service account s nejmenší rolí.
Externí systém mění service_id.Upsert založí duplicity.Držet service_id jako trvalý business klíč.
Import pipeline ignoruje dry-run warnings.Readiness a ref data se rozbijí až po commitu.V CI/pipeline blokovat critical warnings podle pravidel organizace.
API consumer předpokládá, že C3 service = catalogue service.Špatné mapování integrace.Rozlišit C3 service entity a service_catalog service.
Direct DB write.Obchází validaci, audit a cache.Používat API; DB write jen migračně a s auditním záznamem.
Neukládá se response body při chybě.Nelze zjistit, které pole selhalo.Logovat status, endpoint, request id/correlation a response body.
