08 Administrace

Admin panely, reference data, uživatelé a konfigurační scénáře

Administrace řídí, kdo může s aplikací pracovat, jaké hodnoty se nabízejí ve formulářích, jak funguje SSO, jak se spravují C3 builder data, jak se kontroluje instalace a jak se dokládá auditní stopa.

Teoretický význam administrace

Administrace je control plane aplikace. Pokud jsou špatně nastavené role, reference data, SSO, skupiny nebo importní profily, nebude správně fungovat ani katalog služeb. Admin proto nespravuje jen technické nastavení, ale také governance slovník, podle kterého se hodnotí služby, capability, C3 vazby, readiness a audit.

Dobrá administrace vytváří bezpečný rámec: uživatelé mají nejmenší nutná oprávnění, ref tabulky jsou stabilní, změny jsou auditované, SSO je předvídatelné, importy jsou dohledatelné a C3/FMN evidence má jasný zdroj.

Procesní vazba: Administrace podporuje ITIL governance, information security management, service configuration management a knowledge management. TOGAF doplňuje správu architektonických referencí, capability builderu a datového slovníku. V bezpečnostním řízení odpovídá principům least privilege, traceability a separation of duties.

Web a SSO nastavení

Web/SSO nastavení na /administration/web ovlivňuje přihlášení. SSO režim typicky funguje přes reverse proxy, která aplikaci předává důvěryhodné HTTP hlavičky. Pokud proxy neodstraňuje podvržené hlavičky od klienta, SSO je bezpečnostní riziko.

Administrátorské scénáře

A

Nový reviewer nevidí svoje úkoly

usersemailreviews
  1. Otevřete /administration/users a najděte účet.
  2. Zkontrolujte email, display_name, is_active a roli.
  3. Ověřte, zda governance_review.assigned_to odpovídá e-mailu.
  4. Pokud jde o AD účet, zkontrolujte external_principal.
B

Import hlásí neznámý portfolio kód

ref dataimportcontract
  1. V import review najděte import_issue a field_name.
  2. Otevřete /administration/catalogue-ref a tab ref_PortfolioGroup.
  3. Přidejte kód jen pokud je opravdu legitimní.
  4. Spusťte dry-run znovu a zkontrolujte, že warning zmizel.
C

Zapnutí SSO

SSOproxysecurity
  1. Nejprve nastavte reverse proxy tak, aby odstraňovala podvržené klientské hlavičky.
  2. V /administration/web vyplňte SSO header keys.
  3. Ověřte testovací AD účet s external_principal.
  4. Zapněte auth.sso.enabled až po testu.
  5. Nechte si jeden local admin recovery účet.
D

Nová spiral baseline

FMNC3coverage
  1. V admin/C3 reference nebo capability builderu připravte spiral code, title a status.
  2. Naimportujte C3/FMN data se správným spiral_code.
  3. Ověřte /spirals, capability map a coverage.
  4. Pokud se baseline má stát aktivní, proveďte aktivaci jako governance rozhodnutí.
