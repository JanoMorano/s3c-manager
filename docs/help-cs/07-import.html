07 Import

Import dat, dry-run, validace a audit

Import slouží k naplnění katalogu služeb, C3 taxonomy, capability builderu a FMN spiral dat z externích zdrojů. Každý import musí projít parsingem, normalizací, řešením referencí, dry-run kontrolou a auditovatelným commitem.

Teorie importního procesu

Import není jen upload souboru. Je to řízený ETL proces: parse → normalize → taxonomy resolve → dry-run → validate → upsert → audit → review. Dry-run má běžet server-side, protože pouze server zná databázi, referenční integritu, existující služby, C3 UUID, duplicate rules, importní hash a pravidla pro stub služby.

S3C Manager má importní část navrženou tak, aby zdrojová data nepoškodila governance model. Neznámý číselník nemá potichu vytvořit špatnou hodnotu, chybějící service ID nemá založit nečitelnou službu a C3/FMN import musí být dohledatelný podle spiral baseline a zdrojového souboru.

Procesní vazba: Import podporuje ITIL information and technology dimension, service catalogue management, service configuration management, knowledge management a continual improvement. TOGAF/ArchiMate kontext je důležitý u XML/EA importů: modelovací zdroj se převádí do governance evidence, ne do plnohodnotného EA repository.

Admin Import a reparse

Admin import na /administration/import je hlubší kontrola. Používá se, když běžný import review nestačí: raw relations se nepřevádějí, raw fields nejsou správně interpretované nebo je potřeba provést reparse bez ručního zásahu do databáze.

Batches tabGovernance audit sekce importů a jejich detail.Audit a dohledání chyb.Staré batch záznamy mohou být archivované podle retention.
Raw relaceNezpracované relationship řádky.Po reparse mohou vzniknout nebo změnit service_relation.Reparse mění data, proto vyžaduje admin/edit oprávnění.

Co dělat po importu

  1. Otevřete /import a vyberte poslední batch.
  2. Zkontrolujte počty inserted, updated, failed, warnings a import quality report.
  3. Pokud vznikly stub služby, otevřete jejich detail a doplňte je v editoru.
  4. Otevřete /services/list a ověřte, že služby mají správný status, typ a portfolio.
  5. Otevřete /operations/readiness a zkontrolujte nové blockery.
  6. U C3/FMN importu otevřete /c3/list, capability map, /capabilities?view=coverage a /c3/capability-map-spiral7.
  7. Pokud se import dotýká kritických služeb, zapište decision nebo otevřete governance review.
