# V1.2.1 Release Notes

S3C Manager v1.2.1 is a documentation, localization, and navigation-polish release for the current handover baseline. It keeps the reduced product boundary intact and focuses on acceptance-readiness: clearer Czech/English UI text, clickable decision metrics, portfolio/task drilldowns, service graph usability, and versioned documentation for commit and release preparation.

## Highlights

- Updated active documentation version references to `1.2.1`.
- Synchronized the generated UI text catalogue so the i18n audit reports `missing = 0` and `stale = 0`.
- Polished Czech and English UI copy in the shared i18n catalogues, including readiness, governance, capability, graph, portfolio, installer, and operations wording.
- Added or verified clickable KPI/filter drilldowns across dashboard, catalogue, capabilities, operations, readiness, decisions, reviews, portfolio, and personal tasks surfaces.
- Kept `/help`, `/help-cs`, and `/help-en` as the canonical help entry points and preserved `docs/help-*` as static manual sources.
- Preserved the established reduction boundary: no new reduction pass is introduced in v1.2.1.

## User-Facing Fixes

- Review KPI cards on `/operations/reviews` now open filtered review queues for open reviews, in-review items, and overdue reviews.
- Czech label `V revize` is corrected to `V revizi`.
- English UI no longer exposes Czech leftovers such as `CSV export selhal` or Czech action-label aliases in generated text entries.
- Czech UI wording is more natural for readiness blockers, capability mappings, graph relations, owner filters, data-quality fixes, and installer messages.

## Verification Scope

The v1.2.1 readiness check should include:

- `npm --prefix frontend run lint:i18n`
- `npm --prefix frontend run lint`
- `npm --prefix frontend run build`
- Browser smoke for `/operations/reviews` KPI filters.
- Spot-check of `/help`, `/help-cs`, `/help-en`, `/catalogue`, `/capabilities`, `/operations`, `/operations/readiness`, `/operations/decisions`, `/portfolio`, and `/cockpit/my-tasks`.

## Compatibility Notes

- No retired API compatibility window is reintroduced.
- Removed API surfaces remain documented in `docs/api-deprecations.md`.
- Legacy frontend aliases and `/admin/*` routes remain outside the final application surface.
- Runtime package metadata may still require a separate build/version bump if the release process expects package files, Docker tags, or installer metadata to advertise `1.2.1` directly.
