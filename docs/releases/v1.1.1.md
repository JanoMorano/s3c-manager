# V1.1.1 Release Notes

## Topic

Service Governance Cockpit and Dynamic C3 Capability Maps

## Short Description

S3C Manager v1.1.1 finalizes the governance cockpit release line with dynamic C3 capability map creation, clearer Operations navigation, version metadata cleanup, and the removal of the legacy FMN Air C2 production page.

## What Changed

### C3 Capability Maps

- C3 Capability Builder can create new capability map baselines such as `Spiral 99`.
- Custom maps are available through dynamic routes such as `/c3/capability-map-spiral99`.
- Capability map titles are stored per spiral and default to `C3 Taxonomy Catalogue — Baseline <number>`.
- Builder items are saved under the selected spiral instead of being limited to Spiral 6 or Spiral 7.

### C3 Dashboard

- The `Top Parent Capabilities` chart now supports click drilldown.
- Clicking a parent capability opens an explanation dialog with the number of related items.
- The dialog links to the filtered C3 catalogue, including root capabilities.

### Operations Cockpit

- Operations is split into focused tabs:
  - `Health`
  - `Governance`
  - `Pricing`
  - `Owners`
  - `C3 mapping`
- Each tab groups the relevant evidence, queues, coverage indicators, and action links.
- Owner load, contract overlap, renewal risk, governance advisor, pricing gaps, and C3 mapping gaps are easier to scan and use during review meetings.

### Product Versioning

- Application/runtime version is aligned to `1.1.1`.
- Sidebar brand no longer embeds the version number.
- Current version is displayed above Help as `v.1.1.1`.
- Docker Compose, Portainer stack defaults, install fallback metadata, package manifests, and platform seed metadata were updated.

### Production Cleanup

- The legacy `/c3/fmn-air-c2` page was removed from the frontend production route list.
- Production navigation continues to use generic C3 capability and capability map pages.

### Documentation

- `CHANGELOG.md`, `README.md`, admin/user docs, and operations docs were aligned with the `1.1.1` release.
- Operations documentation now includes a lab load-test snapshot:
  - 30 parallel users for 30 seconds: stable, 0 errors, p95 35.5 ms
  - 100 parallel users for 30 seconds: stable, 0 errors, p95 38 ms
