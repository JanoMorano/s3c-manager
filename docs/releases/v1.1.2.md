# V1.1.2 Release Notes

## Topic

Service Relationship Studio and Capability Governance UX

## Short Description

S3C Manager v1.1.2 turns service, C3, and capability views into a shared manager/admin decision experience. It keeps the existing data model and detailed tabs, but adds readable relationship summaries, admin action queues, impact stories, and release-aligned runtime metadata.

## Executive Summary

- Added a Service Relationship Studio to service detail pages.
- Added a C3 Board summary for map coverage, unmapped evidence, import sync drift, and validation findings.
- Added capability relationship views for the capability hub and capability detail pages.
- Expanded frontend route documentation under `docs/desctiption/`.
- Aligned package, runtime, Docker, Portainer, install, platform seed, README, and visible UI version metadata to `1.1.2`.

## What Changed

### Service Views

- Service detail now starts with a relationship studio:
  - manager-readable service summary
  - readiness/completeness metric
  - dependency and C3 link metrics
  - admin action queue
  - requester, owner, support, and capability relationship map
  - key operational facts
- Existing service tabs remain available for overview, offerings, request/support, coverage, and governance.

### C3 Board

- C3 dashboard now has a board-style summary before the detailed charts.
- The board explains:
  - mapped coverage
  - unmapped evidence queue
  - import sync drift
  - review and link validation findings
  - taxonomy-to-impact story from imported item to capability, evidence, and spiral.

### Capability Views

- Capability hub now explains capabilities as a manager-readable map instead of only a list.
- Capability detail now highlights:
  - coverage confidence
  - service evidence count
  - uncovered gaps
  - overlap candidates
  - evidence document count
  - next admin actions.

### Documentation and Versioning

- Added generated frontend route descriptions under `docs/desctiption/`.
- Updated release highlights and documentation examples for `1.1.2`.
- Updated package metadata and deployment/runtime defaults for `1.1.2`.
