# V1.1.2 Release Notes

## Topic

Layout v2 Governance Cockpit and Service 360 UX

## Short Description

S3C Manager v1.1.2 turns the application into a clearer governance and decision cockpit for services, capabilities, C3/FMN relationships, readiness, reviews, impact, and administration. The release keeps the mockup-driven visual direction from `layout-mockup.html` and `layout-mockup-v2.html`, while tightening the data and process behavior underneath it.

## Executive Summary

- Updated the global shell, sidebar, top bar, page headers, typography, KPI blocks, and cockpit surfaces according to the Layout v1/v2 proposals.
- Added and refined Service 360, Relationship Studio, Readiness Gate, Reviews Kanban, Decision Log, Impact Analysis, Import Workspace, Graph Workspace, C3 entity workspace, and Administration Users views.
- Added governance contracts for per-user notifications, user preferences, service requests, readiness explanations, and C3 board state through migration `28_enterprise_governance_contracts.sql`.
- Preserved the lightweight product boundary: S3C Manager is a governance cockpit between ITSM, EA, service catalogue, and project/service management processes, not a ticketing system or full CMDB.
- Aligned package, runtime, Docker Compose, Portainer, install fallback, `.env.example`, README, changelog, release notes, and visible UI version metadata to `1.1.2`.

## What Changed

### Global Layout

- Sidebar groups follow persona/process intent instead of mixing operational and catalogue concepts.
- Top bar supports breadcrumbs, global search behavior, view-switch behavior outside editors, primary action, and notification entry point.
- Page headers, KPI cards, empty states, loading states, and tokenized styling were normalized across the main governance pages.
- Home is positioned as a daily cockpit instead of a simple launcher.

### Service Governance

- Service 360 now combines business and technical views, a relationship studio, readiness signal, lifecycle bar, dependencies, C3 mappings, ownership, support path, and audit context.
- Service editor follows the v2 editor pattern: vertical sticky sub-navigation, validation badges, sticky save bar states, read-only Service ID styling, requestable warning banner, publish gate, lifecycle transition rules, and collection editing patterns.
- Service impact analysis now produces visible output after running analysis instead of leaving the user without feedback.

### Operations and Governance

- Readiness Gate was rebuilt to match the mockup-driven explanation model: what is missing, why it matters, how to fix it, and what evidence is expected.
- Reviews Kanban uses modal-driven status changes with rationale, evidence, defer expiry, side-effect-free readiness pre-flight, and decision log preview.
- Decision Log was restored as an audit-facing governance view.
- My Tasks, Owner Load, and Operations Cockpit were adjusted for clearer manager/admin scanning.

### Catalogue, C3, Capability, and Graphs

- Catalogue browsing, service list/detail, capability hub, C3 board, C3 list/detail/edit/code-edit, spiral views, and graph pages were moved closer to the proposed workspace model.
- Graph Workspace uses the three-column pattern: left toolbar, center canvas, collapsible right detail panel.
- Code editor surfaces use a dark editor theme while the surrounding application remains light.

### Administration, Import, and Search

- Administration Users now follows the split layout with KPI counters for total, active, local, and AD/SSO-backed users.
- Import Workspace was reworked toward the v1 wizard model with source selection, dry-run/validation, preview, and commit-oriented flow.
- Search supports recent and saved searches, with a clearer split between quick command search and the full `/search` page.
- New Service onboarding uses a horizontal step indicator again for readability.

### Documentation and Versioning

- Expanded frontend route descriptions under `docs/desctiption/`.
- Updated README, changelog, upgrade guide, installation guide, and this release note for `1.1.2`.
- Updated package metadata and deployment/runtime defaults for `1.1.2`.
- Updated `.env.example` so fresh installs advertise the correct application version.

## Upgrade Notes

- Back up PostgreSQL before upgrading.
- Deploy image/tag `v1.1.2` or set `APP_VERSION=1.1.2` for local Compose builds.
- Ensure migration `28_enterprise_governance_contracts.sql` is applied after startup or through the install/upgrade wizard.
- After upgrade, verify `/api/v1/install/status`, `/api/v1/install/summary`, `/operations/readiness`, `/operations/reviews`, `/operations/decisions`, `/services/impact`, `/services/graph`, `/import`, and `/administration/users`.
