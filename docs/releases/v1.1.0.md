# V1.1.0 Release Notes

## Topic

Capability-Driven Service Portfolio Cockpit

## Short Description

S3C Manager v1.1.0 moves the product from a record-centric service catalogue to a user-driven decision cockpit for service owners, capability managers, and administrators. The release connects Service Catalogue records, C3 taxonomy, FMN spirals, capability coverage, service dependencies, import evidence, and consolidation signals into one navigable operating model.

## What Changed

### User Experience

- Unified Mantine-like dashboard shell across Catalogue, Operations, C3 Dashboard, Capabilities, and Spirals views.
- Decision-first pages with KPI cards, tabs, evidence lists, readiness boards, dependency flow, and consolidation views.
- Sidebar reorganized by user journey: command centre dashboards, service catalogue, C3 reference, FMN spirals, and administration.
- Theme support for light/dark/system tokens.
- Language and persona settings live in the user information area; the quick user dropdown is intentionally limited to profile and logout actions.

### Internationalization

- Shared i18n catalog supports `cs`, `en`, `sk`, and `de`.
- Locale aliases such as `cze`, `svk`, `ger`, and `deu` normalize to canonical locale codes.
- Primary navigation, dashboards, C3 list controls, and service editor C3 mapping preview use shared i18n keys.

### Capability and FMN Architecture

- Generic Level-3 capability coverage engine replaces route-specific coverage payload builders.
- Production navigation uses the generic capability pages and endpoints.
- Coverage, gaps, duplicate coverage, overlap, consolidation candidates, and evidence documents are served through generic capability endpoints.
- Coverage views no longer expose developer-local PDF paths or route-specific hardcoded requirement constants.

### Service Owner Workflow

- Service editor C3 mapping section loads Level-3 capabilities and previews coverage impact before save.
- Preview shows affected spirals, newly covered requirements, and duplicate coverage risk.
- End-to-end regression covers preview → save → capability dashboard confirmation.

### Import and Data Readiness

- NATO import smoke covers all files in `import-nato`.
- ArchiMate XML, CSV, XLSX, and JSON import routes are spiral-aware where applicable.
- Bulk import wizard supports dry-run planning, blocker/ready indicators, spiral override controls, guarded commit, and run-result summary.

## Release Acceptance Focus

The release is ready for broader acceptance when these checks are green on a local runtime with real imported NATO/C3 data:

- all backend Jest suites that are not environment-blocked pass
- frontend TypeScript and style lint pass
- Docker image builds and app container starts
- NATO import smoke imports every file without failed files or row errors
- local-account Playwright flows pass for login, navigation, user menu, service edit, import, service detail, and mapping preview
- browser smoke confirms the main dashboards render without `500`, `404`, or client-side console errors
