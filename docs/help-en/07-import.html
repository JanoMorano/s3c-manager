07 Import

Data import, dry-run validation and audit

Import brings service, reference, C3, FMN and integration evidence into the application. The safest workflow is profile selection, file upload, dry-run preview, issue review and then controlled execution.

Import process theory

Import is a governed data intake process, not a blind upload. The application should show what will be created, updated, skipped or rejected before data is written to PostgreSQL.

Dry-run report

A dry-run should be read before execution. Focus on unknown reference values, duplicate service IDs, missing owner, unmapped C3 capability, invalid lifecycle value and rows that would overwrite existing data.

Post-import actions

After import, open the service list, readiness queue, capability coverage and import audit. Imported evidence often needs owner confirmation, governance review or manual cleanup before publication.
