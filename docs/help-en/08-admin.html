08 Administration

Administration panels, reference data, users and configuration scenarios

Administration controls the safe operation of the instance: users, groups, roles, SSO, logs, reference data, import profiles, capability builder and installation diagnostics.

Why administration matters

Good administration keeps the catalogue understandable and secure. Poor reference data or roles can confuse users, hide tasks from reviewers, break imports or expose functions to the wrong audience.

Roles

•User

Can browse the catalogue and consume readable service information.

•Content admin / editor

Can maintain service and architecture content.

•Administrator

Can manage users, roles, reference data, imports and instance settings.

Common scenarios

•New reviewer cannot see tasks

Check role, group membership, preferred language and assigned review ownership.

•Import reports unknown portfolio code

Add or correct reference data before re-running import.

•Enable SSO

Configure trusted headers or proxy integration, then test fallback local admin access.

•New spiral baseline

Use capability builder and verify it appears in capability maps and import baselines.
