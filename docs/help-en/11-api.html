11 API

API parameters and integration scenarios

The API lets other tools read catalogue, service, C3, governance and import context. It should be used with RBAC, authenticated sessions and clear ownership of integrations.

General API rules

Use versioned endpoints, authenticated request links, least-privilege roles, stable query parameters and safe error handling. Retired endpoints should return explicit 410 responses until their compatibility window expires.

Integration scenarios

•ITSM portal

Reads requestable service offerings and links users to the correct request channel.

•EA repository

Exchanges capability and architecture evidence without replacing the modelling tool.

•Reporting dashboard

Reads readiness, owner load, portfolio and decision metrics.

•Import pipeline

Pushes validated service or C3 evidence through dry-run and controlled execution.

Common integration mistakes

Do not hard-code retired endpoints, bypass RBAC, parse user-facing labels as identifiers or ignore pagination. For write integrations, always test dry-run and audit output before enabling production execution.
