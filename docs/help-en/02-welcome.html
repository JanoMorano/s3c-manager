02 Application overview

What S3C Manager is and where it fits in enterprise governance

S3C Manager is best used as a governance-first cockpit above services, capabilities, C3/FMN relations and readiness. It is not a ticketing system, monitoring platform, full CMDB or replacement for an Enterprise Architecture repository.

Short definition

The application answers practical portfolio and governance questions: what services exist, who owns them, which capabilities and C3 elements they support, whether they are ready for publication or change, where coverage gaps exist and what will be affected by a change.

What it is and what it is not

•Governance cockpit

Decision, readiness, ownership, exception and review context in one place.

•Service catalogue

A managed catalogue of services, offerings, owners, lifecycle state and consumer-facing context.

•Capability and C3/FMN cockpit

Mapping between services, business capabilities, C3 capabilities, spirals, applications and data objects.

•Not ticketing or monitoring

It references operational readiness and decisions, but does not replace incident queues or live telemetry.

•Not a full CMDB

It keeps service-relevant dependencies and ownership, not every infrastructure configuration item.

•Not an EA repository

It supports architecture mapping without becoming a full ArchiMate modelling repository.

Best use

Use S3C Manager when an organization needs a shared decision surface between service management, architecture, ITSM and project delivery. It is especially useful before service publication, during portfolio review, while resolving capability gaps, and before change or release decisions.

Process place in the organization

•Service intake / demand

Capture a new service or change request with owner, purpose, lifecycle and initial capability context.

•Service portfolio management

Classify services by portfolio, lifecycle, criticality, duplication and risk.

•Architecture and capability mapping

Map services to capabilities, C3 elements, applications, data objects and integrations.

•Service design

Complete offering, SLA, support model, dependencies, reviewers and security attributes.

•Readiness gate

Check whether the service can be published, changed or retired.

•Governance review

Approve, reject, defer or record an exception as an auditable decision.

•Change / release / transition

Use impact analysis before changing a service, capability or C3 item.

•Operate and improve

Track due reviews, warnings, governance queue, owner load, gaps and overlaps.

When not to deploy it

Do not use S3C Manager as the only tool for incident handling, monitoring, detailed asset discovery or complex enterprise modelling. In those cases it should integrate with the specialist system and keep only the governance and catalogue context.
