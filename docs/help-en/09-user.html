09 User

User menu, profile, preferences and roles

The user profile controls personal identity, language, password and role-visible work. It also explains why different users may see different navigation or actions.

User menu

The user menu gives access to profile settings, language preference, password change when local authentication is used, and sign-out.

Profile fields

FieldMeaningImpact
Display nameHuman-readable user name.Used in ownership, reviews and audit context.
EmailContact and future notification address.Helps reviewers and owners identify accountable users.
Preferred languagecs or en.Controls UI language, locale cookie and help link target.
RoleUser, editor/content admin or administrator.Controls navigation and permitted actions.

Language preference and help

When preferred language is Czech, the Help link opens help-cs. When preferred language is English, it opens help-en. The same locale is stored in the sc_locale cookie for server-side redirects.

Common issues

•I do not see my review

Check that the review is assigned to your user or group and that your role permits review work.

•I want a less technical view

Use catalogue, service detail and cockpit first; expert C3 and raw import evidence are meant for administrators and architects.

•I cannot change password

Password change is available for local accounts. SSO or AD accounts are managed by the identity provider.
