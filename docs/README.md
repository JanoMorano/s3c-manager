# S3C Manager Documentation

This directory now keeps only active product, operations, installation, API, and release documentation at the top level.

## Active Documentation

- `help-cs/` - Czech source manual for the in-app Help served from `/help-cs`.
- `help-en/` - English source manual for the in-app Help served from `/help-en`.
- `admin-guide.md` - administrator operating guide.
- `user-guide.md` - user-facing operating guide.
- `installation.md` - installation and runtime setup.
- `upgrade.md` - upgrade and migration guidance.
- `api-deprecations.md` - v1.2 removed API inventory and replacements.
- `domain-model.md`, `modules.md`, `operations.md`, `governance-cockpit.md`, `c3-module.md` - product and architecture references.
- `import-formats.md`, `integrations.md` - import and integration contracts.
- `personas.md`, `product-positioning.md` - product context and target users.
- `img/` - manual image drop area for curated documentation screenshots.
- `releases/` - release notes.

## Archived Planning And Dev Records

- Completed planning, reduction, improvement, navigation-hardening, and audit artifacts are archived under `backups/docs-trash-*`.
- The `backups/` directory is intentionally ignored by Git and Docker build context. Review the archive there before deleting anything permanently.
